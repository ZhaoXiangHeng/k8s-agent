# 工作负载扩缩容

## 步骤
1. 使用 `get_deployments` 或 `get_statefulsets` 查看当前工作负载状态，确认副本数和就绪情况
2. 检查 HPA 配置（如果存在），确认是否需要调整 HPA 而非直接修改副本数
3. 使用 `get_pods` 查看相关 Pod 的资源使用情况，评估是否需要同时调整资源配额
4. 确认目标副本数的合理性（避免过度扩容导致集群资源耗尽）

## 注意事项
- 缩容前确认无活跃连接或正在处理的任务
- 有状态应用需注意缩容顺序和持久卷绑定
- 建议逐次调整，每次观察稳定后再继续

## 输出要求
1. 当前副本数和就绪状态
2. 扩缩容建议及理由
3. 相关风险提示
